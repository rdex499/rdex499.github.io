AD0_VAL Discussion

The AD0_VAL definition determines the least significant bit of the IMU's I2C address. The ICM-20948 supports two possible addresses depending on whether the ADR jumper is bridged:sidenot

  • AD0_VAL = 1 → I2C address 0x69 (ADR jumper not bridged, default)
  • AD0_VAL = 0 → I2C address 0x68 (ADR jumper bridged)

On the SparkFun breakout, the ADR jumper is not bridged by default, so the address is 0x69. The example code already defaults to AD0_VAL = 1, which matches our board setup. 

#define AD0_VAL 1  // default: ADR jumper not bridged → 0x69

Acceleration & Gyroscope Data

The board was rotated, flipped, and accelerated to observe how each sensor responds.

  • Accelerometer: Measures linear acceleration, including gravity. When the board is placed flat on a table, the Z-axis reads approximately +1g (≈1000 mg) while the X and Y axes are near 0g. When rotated 90° onto its side, gravity shifts to the corresponding axis, demonstrating that the accelerometer measures the projection of the gravity vector onto each axis. Small variations are visible even when the board is held still, indicating measurement noise inherent to the sensor.
  • Gyroscope: Measures angular velocity (deg/s). When the board is stationary, all gyro axes read approximately 0 dps. During rotation, the axis of rotation shows a spike proportional to angular velocity, returning to zero once motion stops. Unlike the accelerometer, the gyroscope does not provide absolute orientation.

The small fluctuations observed in both accelerometer and gyroscope readings, even when the board is stationary, are due to sensor noise and minor electrical and mechanical disturbances. This noise motivates the filtering techniques implemented later in the lab.

Flat on table: Z-axis ≈ 1000 mg (gravity), X and Y axes ≈ 0 mg. Gyroscope reads ~0 dps on all axes.

Rotated 90° onto its side: Gravity shifts from the Z-axis to the axis now pointing downward, confirming correct orientation sensing.

Complementary Filter

The complementary filter fuses the accelerometer (accurate long-term, noisy short-term) with the gyroscope (smooth short-term, drifts long-term): 

float alpha = 0.98;  // TODO: tune

float dt = (micros() - last_time) / 1000000.0;
last_time = micros();

pitch = alpha * (pitch + myICM.gyrX() * dt) + (1 - alpha) * pitch_a;
roll  = alpha * (roll  + myICM.gyrY() * dt) + (1 - alpha) * roll_a;
Design Choice: alpha ≈ 0.98 — trusts the gyroscope for short-term changes (98%) and uses the accelerometer to correct drift (2%).

Timestamped IMU Data in Arrays

Stored timestamped IMU readings in global arrays on the Artemis, reusing the batch recording approach from Lab 1B: 

// Global arrays for IMU data logging
#define MAX_DATA_SIZE 500

int time_array[MAX_DATA_SIZE];
float pitch_array[MAX_DATA_SIZE];
float roll_array[MAX_DATA_SIZE];
float yaw_array[MAX_DATA_SIZE];
int data_index = 0;

// In the main loop (no BLE overhead):
if (data_index < MAX_DATA_SIZE) {
    time_array[data_index] = millis();
    pitch_array[data_index] = pitch;
    roll_array[data_index] = roll;
    yaw_array[data_index] = yaw;
    data_index++;
}
